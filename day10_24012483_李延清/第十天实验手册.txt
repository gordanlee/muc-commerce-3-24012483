# 
## 一、今天要完成什么
你不需要默写算法，也不需要调参。你需要让三个模型使用同一份数据完成训练，然后读懂结果、选择模型并输出预测。

## 二、三种模型的直观理解
+ 逻辑回归：综合多个证据，输出流失倾向和最终类别；
+ 决策树：连续提出若干“是或否”问题；
+ 随机森林：让多棵不同的树共同投票。

## 三、公平比较规则
三个模型必须使用同一份训练集、测试集和预处理方法。不能因为某个模型结果不好就重新随机划分数据。

## 四、指标翻译
+ 准确率：全部用户中猜对多少；
+ 流失召回率：真实流失用户中找回多少；
+ 精确率：预测为流失的人中有多少确实流失；
+ FN：真实流失却被漏掉；
+ FP：实际未流失却被误报。

## 五、Notebook任务
1. 填写个人信息；
2. 确认`CustomerID`和`Churn`不进入X；
3. 训练逻辑回归；
4. 训练决策树；
5. 训练随机森林；
6. 比较同一测试集结果；
7. 选择最终模型并说明理由；
8. 输出预测和高风险名单；
9. 保存并重新加载模型；
10. 完成复盘和提交检查。

## 七、重要提醒
高流失概率不代表用户一定会流失。预测只能帮助确定优先关注顺序，不能替代业务核实和人工判断。#

## 不作扣分要求
+ 不要求推导算法公式；
+ 不要求从零编写预处理流水线；
+ 不要求超参数搜索、交叉验证、ROC-AUC或深度学习；
+ 不因无法解释每一个API参数扣分。

## 一票否决项
+ `Churn`进入特征；
+ 测试集参与`fit`；
+ 三个模型使用不同测试集却直接比较；
+ 删除断言或伪造CSV结果；
+ 将预测概率描述为确定事实。
